# RCG /handoff

Create a concise end-of-session handoff so the next person can resume without extra context gathering.

Intent: End-of-session continuity handoff.

## Required inputs
Before proceeding, ensure you have the following. If any are missing, ask the user to provide them:
- Completed work during current session
- Open tasks and owners
- Active blockers and dependencies
- Decisions made and rationale
- Next session constraints (time, deadlines, dependencies)

## Workflow
1. Ask the user what was completed, what's still open, and what's blocked if not already provided.
2. Summarize completed work with concrete outcomes.
3. List open tasks with the first next action for each.
4. Capture active blockers, dependencies, and owners.
5. Record any decisions made and their rationale.
6. Produce a short startup checklist for the next session.

## Constraints
- Keep it brief and execution-oriented.
- Remove stale or low-value reminders.
- Ensure the next person can resume without extra context gathering.

## Exit criteria
- Completed work is specific and outcome-based.
- Open tasks and blockers are clear.
- Startup checklist is actionable.

Paste-ready summary:
```
Done: ...
Open: ...
Blocked by: ...
Next session first step: ...
```
